# 用户旅程索引

用户旅程（User Journey）描述用户与 FFP 交互的端到端场景，按**用户使用产品的时间线**组织。

## 旅程列表

| 编号 | 名称 | 覆盖 Theme | 用户画像 | 说明 |
|------|------|-----------|----------|------|
| [UJ-01](UJ-01-first-time-setup.md) | 首次上手 | TH-01, TH-02 | 新手家长 | 注册 → 创建家庭 → 录入第一笔收入 |
| [UJ-02](UJ-02-daily-record.md) | 日常记账 | TH-01, TH-03 | 家庭管理员 | 收入/支出录入 → 分类调整 → 查看列表 |
| [UJ-03](UJ-03-monthly-review.md) | 月度复盘 | TH-04 | 家庭管理员 | 查看仪表盘 → 分析分类 → 调整预算 |
| [UJ-04](UJ-04-family-collaboration.md) | 家庭协作 | TH-02 | 家庭管理员 | 邀请成员 → 权限管理 → 共享数据 |
| [UJ-05](UJ-05-goal-planning.md) | 目标规划 | TH-05 | 长期规划者 | 设定储蓄目标 → 追踪进度 → 调整策略 |

## 旅程格式说明

每个旅程文件包含以下章节：

| 章节 | 说明 |
|------|------|
| **目标** | 用户通过这个旅程想达成什么 |
| **用户画像** | 主要角色（引用 `user-personas.md`） |
| **旅程阶段** | 按时间线排列的关键阶段 |
| **情绪曲线** | 用户在各阶段的情绪高低 |
| **涉及功能区域** | 关联的 Theme / Epic（不映射具体 Feature） |
| **痛点与机会** | 当前假设的痛点，以及产品可优化的机会 |
| **相关旅程** | 与本旅程衔接的其他 UJ |

## 与 Feature 开发的关系

```text
User Journey (业务输入)
    │
    ▼
Designer 阅读 UJ → 理解场景 → 设计 Feature
    │
    ▼
Feature 文档 (feature.md + us-*.md + uc-*.md)
    │
    ▼
Developer / Tester 实现与验证
```

旅程文件**不引用具体 Feature ID**，因为 Feature 是在 Designer 阶段才细化的。旅程与 Feature 的关联由 Designer 在设计过程中建立。
